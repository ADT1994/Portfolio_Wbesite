Data Analysis Higlights:
Project- Retail Customer Data Analysis & Mexican Resturant Data

Customer Transaction Data : This Data(source: Kaggle) includes day to day transactions of customers, It includes three tables: Customer Data, Product Category, Transactions of 4 years
Mexican Restaurant Data: Data source: (Maven Analytics), this data was provided by maven analytics It includes data of 130 Mexican restaurants and their customers.

Finding if a transaction is a Sale or Return type

Some of the values quantity & Rate column of data set are negative while most of them were positive, This implies that particular transaction is a return type.This criteria was used to check if the transaction was Positive(Sale) or Negative(return). These categeorical features was added to the final data set. Further this data set was converted to new data set grouped by Product category column(Women, men, kids, Bath, Furnishing, etc.) to check which category has more return transactions.

Finding Sales to Return Ratio

Once another categorical feature is added to the data frame and grouped by product category, further analysis can be done to get ratios that tell some story. This new grouped data set also aggregated on counts, to get sales and return counts of each product category. To get the return/sales ratio of different products category, their return count was divided by sales counts.These ratios give light to the fact that women, men, kids , mobiles category has most return( 10-12 returns per 100 sales). Categories like DIY, Personal Appliances, cameras have the least returns per 100 sales Transactions(9-10).

Checking Transaction Day (Weekend(including Friday) or Weekday)

After converting object type Date column to dateTime format, weekday can be extracted using weekday() function of pandas DateTime library. This function returns a day number (1 to 7) .These numbers can be furher used to get day(Monday-Sunday) and accordingly check if the transaction day was weekday or weekend. Friday is included as the weekend.

Analysing Mexico Resturants Rating

Restaurant table and Rating tables were joined using MySQL on Restaurant_ID, Average rating of the restaurant was calculated using groupby function of SQL , to calculate how average rating if getting affected by different services provided by restaurants. Other factors related to customers like drinking level, mode of transportation, occupation of customers , budget were used to understand more about customer .This will help in making more customer-centric investment decision.
