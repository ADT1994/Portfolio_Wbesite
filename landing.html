Data Cleaning Highlights:
Project- Customer Data Analysis & Covid-19 Data

Customer Transaction Data : This Data(source: Kaggle) includes day to day transactions of customers, It includes three tables: Customer Data, Product Category, Transaction of 3 years
Covid-19 Data: Data source: Ourworldindata, this data includes information about all covid infection of 207 country around the world.

Checking for NULL/NA values and replacing/removing them

Although initial Data set about retail store do not contain any null values. After merging different tables through primary keys(Prod_cat_code) creates null values. They will be checked for their weightage and removed from data accordingly.

Null Values After Merging Tables

After merging (Transaction+product_Cat) with customer Tables using customer_id as the primary key, there are around 141 null values. There is no data available for these 141 null values in most of the columns. Droping these rows was the better option.

Cleaning Date column

While trying to extract date features from data, the whole column was object type and not dateTime type. Many of the rows were mixed date style of %dd-%mm-%yyyy & %dd/%mm/%yyyy. It was necessary to convert the whole column into one type of date,before converting it into a dateTime data type. Only then features extraction like month, years, weekend, or weekday of transaction could be done.

Cleaning Covid-19 Data from MySQL & Extracting Important Data

Multiple null values were there they were dropped using the Excel filter function. Important variables aggregated with group function of SQL.
